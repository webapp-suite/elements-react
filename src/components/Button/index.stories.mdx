# Button

With `MDX` we can define a story for `Button` right in the middle of our Markdown documentation.

## Basic

<Canvas>
    <Story
        name="type: default"
        args={{
            children: "Default",
        }}
    >
        {(args) => <Button {...args} />}
    </Story>
</Canvas>

## Type

The `default` and `primary` buttons are used in most cases, you can also use other types in specific scenarios.

<Canvas>
    <Story name="type: primary" args={{ type: "primary", children: "primary" }}>
        {(args) => <Button {...args} />}
    </Story>
    <Story name="type: secondary" args={{ type: "secondary", children: "secondary" }}>
        {(args) => <Button {...args} />}
    </Story>
    <Story name="type: accept" args={{ type: "accept", children: "accept" }}>
        {(args) => <Button {...args} />}
    </Story>
    <Story name="type: warning" args={{ type: "warning", children: "warning" }}>
        {(args) => <Button {...args} />}
    </Story>
    <Story name="type: danger" args={{ type: "danger", children: "danger" }}>
        {(args) => <Button {...args} />}
    </Story>
    <Story name="type: text" args={{ type: "text", children: "Text" }}>
        {(args) => <Button {...args} />}
    </Story>
</Canvas>

## Size

Besides default `md` size, Button component provides three additional sizes for you to choose among different scenarios.

<Canvas>
    <Story
        name="size: micro"
        args={{
            size: "micro",
            children: "Button",
        }}
    >
        {(args) => <Button {...args} />}
    </Story>
    <Story
        name="size: macro"
        args={{
            size: "macro",

        }}
    >
        {(args) => <Button {...args} />}
    </Story>
</Canvas>
